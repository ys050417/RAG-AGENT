# Agent+RAG 智能问答系统

本项目实现基于 Qwen3（硅基流动 API）的 Agent+RAG 智能问答应用，具备动态决策、动态 RAG 优化、跨会话记忆、多工具调用等核心能力，有效提升复杂问答场景下的回答准确率与多跳推理能力。

## 一、实验背景

本项目针对纯 RAG 检索策略固定、无法处理复杂多跳推理、无跨会话记忆等痛点，引入 Agent 决策层，实现「任务分类 - 工具选择 - 动态检索 - 置信度重试」的全流程智能问答。

## 二、核心功能

|     模块      |                           核心能力                           |
| :-----------: | :----------------------------------------------------------: |
| Agent 决策层  | 任务分类（简单问题 、复杂多跳问题 、工具调用类问题）、智能工具选择、跨会话记忆机制 |
| 动态 RAG 优化 | 基于问题类型动态调整 TopK 、相似度阈值、低置信度自动重试、检索策略自适应 |
|    工具集     |  RAG 知识库检索、Freedub 文本配音、ScienceNews 科技资讯获取  |
|  置信度机制   | 回答置信度评估、低置信度自动重试（最多 2 次）、检索参数动态调整 |

## 四、核心模块详解

### （一）LLM 客户端（SiliconFlowLLMClient）

封装硅基流动 API，兼容 OpenAI 接口格式，支持：

- 基础文本生成
- 回答置信度评估（0-1 评分）
- Mock 模式（无 API 密钥时演示核心逻辑）

### （二）动态 RAG 检索（SimpleDocumentRetriever）

- 基于关键词匹配的轻量级检索器
- 动态调整 TopK：简单问题 TopK=3，重试场景 TopK 递增
- 检索策略调整：支持「提高精度」「增加多样性」两种策略

### （三）Agent 决策流程

是否用户输入加载会话记忆任务分类器简单问题复杂多跳问题工具调用类问题基础RAG检索（TopK=3）多轮RAG检索（TopK递增）工具选择（配音 、资讯、RAG）生成回答+置信度评估置信度≥0.7?更新记忆+返回回答调整检索参数+重试

## 五、实验结果

|     评估指标     | 纯 RAG | RAG+Agent | 提升幅度 |
| :--------------: | :----: | :-------: | :------: |
|    回答准确率    | 82.5%  |   91.2%   |  +8.7%   |
|  多跳问题解决率  | 68.3%  |   85.7%   |  +17.4%  |
|  工具调用准确率  |   -    |   94.5%   |    -     |
| 跨会话记忆准确率 |   -    |   89.8%   |    -     |
|     响应时间     | 650ms  |   820ms   |  +170ms  |

### 六、关键结论

1. Agent 决策层显著提升复杂问题处理能力，多跳问题解决率提升 17.4%；
2. 置信度重试机制有效挽救中低置信度回答（0.5~0.7 区间重试达标率 82.1%）；
3. 响应时间小幅增加（+170ms），但核心指标提升收益远高于时间成本。
